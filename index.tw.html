 提示 API

 目的


Prompt API 是為了本地實驗而提供的，以促進內建 AI 用例的發現。使用此 API，你可以將自然語言指令發送到 Chrome 中的 Gemini Nano 實例。


雖然 Prompt API 提供了最大的靈活性，但它不一定能提供最佳的結果，在某些情況下可能無法提供足夠的品質。這就是為什麼我們相信，與微調或專家模型配對的特定任務 API（例如 Translation API）將能提供顯著更好的結果。我們對 Prompt API 的期望是，它能幫助加速發現引人注目的使用案例，以便為特定任務 API 的路線圖提供資訊。

 API 概述

 範例程式碼

 立即版本

// Start by checking if it's possible to create a session based on the availability of the model, and the characteristics of the device.

const canCreate = await window.ai.canCreateTextSession();

// canCreate will be one of the following:

// * "readily": the model is available on-device and so creating will happen quickly

// * "after-download": the model is not available on-device, but the device is capable,

//   so creating the session will start the download process (which can take a while).

// * "no": the model is not available for this device.

if (canCreate !== "no") {

  const session = await window.ai.createTextSession();

  // Prompt the model and wait for the whole result to come back.  

  const result = await session.prompt("Write me a poem");

  console.log(result);

}

 串流版本

const canCreate = await window.ai.canCreateTextSession();

if (canCreate !== "no") {

  const session = await window.ai.createTextSession();

  // Prompt the model and stream the result:

  const stream = session.promptStreaming("Write me an extra-long poem");

  for await (const chunk of stream) {

    console.log(chunk);

  }

}

 Session 選項


每個會話可以使用 topK 和 temperature 進行自訂。這些參數的預設值從 window.ai.defaultTextSessionOptions() 返回。

const defaults = await window.ai.defaultTextSessionOptions();

const session = await window.ai.createGenericSession(

    {

      temperature: 0.6, 

      topK: defaults.topK

    }

);

 終止會話


如果您不再需要會話，請呼叫 destroy() 來釋放資源。當會話被銷毀後，它將無法再被使用，且任何正在進行的執行都會被中止。如果您打算經常提示模型，您可能會希望保留會話，因為建立會話可能需要一些時間。

await session.prompt(`

  You are a friendly, helpful assistant specialized in clothing choices.

`);

session.destroy();

// The promise will be rejected with an error explaining that the session is destroyed.

await session.prompt(`

  What should I wear today? It's sunny and I'm unsure between a t-shirt and a polo.

`);

 控制序列


字元序列 <ctrl23> 是 Gemini Nano 的控制序列。它幫助模型理解模擬互動的回合以及何時應該回應。該序列旨在直接用於提示中（詳情請參見提示 101 部分）。

💡


目前，控制序列 <ctrl23> 會自動由 Prompt API 在提示的結尾處添加.


然而，對於一次性*或少量**提示，最好在每次模擬互動（shot）結束時添加控制序列。請參閱這些範例.


一次性: 一個提示，其中模型僅提供單一範例來學習並生成回應。


少量示例: 提供模型一些範例，通常範圍從兩到五個，以幫助其生成更準確的回應。

 例外


Prompt API 可能會從 AI 執行階段接收到錯誤。請參閱本節以獲取可能的錯誤列表，以及它們如何映射到 DOMExceptions。

 注意事項


canCreateTextSession() 的 "after-download" 狀態


“after-download” 狀態和行為不受支援。API 不會觸發模型的下載。相反，Chrome 會觸發下載，這可能是 chrome://flags 狀態變更的一部分，或者是因為另一個裝置上的 AI 功能。

Streaming


目前，promptStreaming() 返回一個 ReadableStream，其區塊依次建構在彼此之上。


例如，以下程式碼記錄了一個序列，例如 "Hello," "Hello world," "Hello world I am," "Hello world I am an AI."

for await (const chunk of stream) {

  console.log(chunk);

}


這不是預期的行為。我們打算與平台上的其他串流 API 對齊，其中的區塊是單個長串流的連續部分。這意味著輸出將是一個序列，如 "Hello", " world", " I am", " an AI".


目前，為了達到預期的行為，你可以實作以下內容:

let result = '';

let previousLength = 0;

for await (const chunk of stream) {

  const newContent = chunk.slice(previousLength);
 console.log(newContent);

  previousLength = chunk.length; 

  result += newContent;

}

console.log(result);


會話持久性和複製


雖然有一個叫做 AITextSession 的物件，但持久性尚未實現。任何發送到模型的資訊都不會被保留，這意味著每次都必須重新發送整個對話。換句話說，每次呼叫 prompt() 或 promptStreaming() 都是獨立的。


一旦實現持久性，我們打算允許會話複製。這意味著可以設置具有基線上下文的會話並提示模型，而無需重複。

 隱身模式和訪客模式


我們對 Prompt API 的實作目前不支援無痕模式，也不支援訪客模式。這是由於 AI 執行層中的相依性，而不是永久性的限制。

 企業


如果將 GenAILocalFoundationalModelSettings 設定為“Do not download model”，此 API 將無法運作。

 工人


截至 2024 年 6 月 18 日，Prompt API 可在 Dedicated Workers、Shared Workers 和 Service Workers 中使用。對於後兩種類型的 workers，你將需要最新的 Chrome Canary 或 Dev 版本。

DOs

DONTs


包括範例來指導您的提示。一個範例，或一次提示，比沒有範例要好。少量提示在指導模型返回您預期的結果方面是最好的。


避免使用有正確或錯誤答案的用例。

該模型可能太小而無法正確回答知識問題，並且在需要獲得完美答案的任務中也可能會遇到困難。設計您的功能或用戶體驗時，請考慮這些不完美之處。


添加規則。你可以通過在提示中添加規則來提高輸出的品質。


範例: "以友善的語氣回應", "如果不清楚，使用'模糊'類別", "不要提供任何解釋。"


避免繞過使用者的使用案例。LLMs 可能並不總是有令人滿意的回應。因此，最好將您的 AI 功能定位為支持使用者完成任務的工具（例如，生成使用者可以快速查看和調整的潛在關鍵字列表）。


新增一個角色。這可以幫助模型返回更符合您需求的內容。


範例: “你剛買了一個產品，並且正在為該產品撰寫評論。請重新措辭 [...]”


避免自訂參數。雖然可以更改參數（例如溫度和 topK），但我們強烈建議您保持預設值，除非您已經沒有提示的想法或需要模型表現得不同。

 設定 Phi-3 物件的佇列和堆疊資訊。呼叫程式碼片段來執行函式庫的原理圖。建構和設定套件的影片指南。使用 for 迴圈來建立類別。平行處理和交易式程式碼產生器。任意類別的延展性和相依套件。相依性注入和保留字 Metadata。複製記憶體的內建全域相容性。重新整理函式來建立文件範例和展示。品質指南和秘訣


指定輸出語言。如果你用英文提示模型但需要非英文回應，請在提示中指定輸出語言。


範例: “請用西班牙語重新表述以下句子。Hola, cómo estás”。

 輸出品質


目前 Prompt API 的實作主要是用於實驗，當與我們打算發布的任務 API 整合時，可能不會反映最終的輸出品質。


也就是說，如果您看到模型生成有害內容或有問題的回應，我們鼓勵您分享對輸出品質的反饋。您的報告對於幫助我們改進和提升我們的模型、API 和 AI 執行層的元件至關重要，以確保安全和負責任的使用。


是否有方法知道輸入提示的token長度？


目前沒有。我們承認這對拉丁語系的語言來說是不方便的，考慮使用這個經驗法則：一個token大約是四個字符長。


上下文視窗的長度是多少?


預設的上下文視窗設置為 1024 個標記。對於 Gemini Nano，理論上的最大值是 32k，但上下文大小和性能之間存在權衡。.


當提示中的標記數量超過上下文窗口時會發生什麼？


目前的設計僅考慮給定輸入中的最後 N 個標記。因此，在提示中提供過多的文本，可能會導致模型忽略提示的開頭部分。例如，如果提示以"Translate the following text to English:..."開頭，並在其前面有數千行文本，模型可能無法接收到提示的開頭部分，從而無法提供翻譯。


某些提示在使用流執行時會停止。


讓我們知道您是否能重現該問題: 提示、會話選項、關於設備的詳細資訊等。同時，重新啟動 Chrome 並重試。


最後手段故障排除

 替代步驟


一些參與者報告說，以下步驟幫助他們顯示該元件:


  • 嘗試將 chrome://flags/#optimization-guide-on-device-model 更改為 Enabled 而不是 Enabled BypassPerfRequirement，然後重試設定步驟    .

 模型下載延遲


瀏覽器可能不會立即開始下載模型。如果您的電腦符合所有要求，並且在呼叫 window.ai.createTextSession() 後，您沒有看到模型在 chrome://components 上開始下載，且 Optimization Guide On Device Model 顯示版本 0.0.0.0 / New，請保持瀏覽器開啟幾分鐘，等待排程器開始下載。

 除錯日誌

 如果一切都失敗:

  1.  打開新分頁
  2.  前往 chrome://gpu
  3.  下載 gpu 報告

  4. 前往 chrome://histograms/#OptimizationGuide.ModelExecution.OnDeviceModelInstallCriteria.AtRegistration.DiskSpace

partial interface WindowOrWorkerGlobalScope {

interface AI {

  Promise<AIModelAvailability> canCreateTextSession();

  Promise<AITextSession> createTextSession(
     optional AITextSessionOptions options = {});

  Promise<AITextSessionOptions> defaultTextSessionOptions();

[Exposed=(Window,Worker)]

interface AITextSession {

  Promise<DOMString> prompt(DOMString input);

  ReadableStream promptStreaming(DOMString input);

  undefined destroy();

  AITextSession clone(); // Not yet implemented (see persistence and cloning for context)

};

dictionary AITextSessionOptions {

  [EnforceRange] unsigned long topK;

  float temperature;

};

enum AIModelAvailability { "readily", "after-download", "no" };
